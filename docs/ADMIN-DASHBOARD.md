## ארבעת הטאבים

### 1. סקירה (Overview)
תמונת מצב כללית במבט אחד:
- **כרטיסי KPI:** סה"כ סריקות · ב-24 שעות · ב-7 ימים · ב-30 יום · דומיינים ייחודיים · ציון ממוצע · דומיינים מאומתים · לידים · נחסמו ע"י WAF (+ כמה חולצו ע"י stealth) · סריקות שנכשלו · ריצות Stage 2 · סריקות מוצלחות.
- **משפך המרה:** ביקורים בדף → התחילו סריקה → השלימו סריקה → Stage 2 התחילו → Stage 2 הושלמו.
- **התפלגות ציונים:** כמה אתרים קיבלו A / B / C / D / F (סריקות מוצלחות בלבד).
- **הממצאים הנפוצים ביותר:** הבעיות שהכי חוזרות (הממצא המוביל בכל סריקה), עם צבע לפי חומרה.
- **מדינות ו-WAF:** מאיפה סורקים (לפי IP), ואילו ספקי WAF זוהו.
- **דומיינים שנסרקו הכי הרבה:** כולל סריקות חוזרות, ציון אחרון וזמן אחרון. כל דומיין לחיץ לסריקה חוזרת.

### 2. סריקות (Scans)
כל מי שסרק — מתוך טבלת `vs_scan_log`:
- זמן · דומיין (לחיץ לסריקה חוזרת) · ציון + דירוג · ממצא מוביל (עם חומרה) · ספק WAF · מדינה · תוצאה.
- **חיפוש** לפי דומיין + **סינון** לפי תוצאה (הצליחו / נחסמו ע"י WAF / נדחו ע"י האתר / לא נגישים).
- במובייל: כרטיסים נוחים במקום טבלה שגוללת לצדדים.
- מציג עד 500 הסריקות האחרונות.

### 3. לידים (Leads)
שני סוגי לידים במקום אחד — מתוך טבלת `vs_leads`:
- **פניות יצירת קשר** — מטופס "צור קשר" באתר (שם, מייל, הודעה).
- **אימותי בעלות** — מי שעבר Stage-3 ונתן מייל (הליד הכי חם: בעל דומיין שרוצה סריקה עמוקה). כולל הדומיין, שיטת האימות, והאם אומת.
- לכל ליד: צ'יפ מקור + סטטוס, מייל לחיץ (mailto), דומיין לחיץ, וכפתורי **ניהול סטטוס** (נקרא / נענה / אורכב).
- **סינון** לפי מקור + הצגת/הסתרת מאורכבים. מספר הלידים החדשים מופיע כ-badge על הטאב.
- בתחתית: טבלת **דומיינים מאומתים** (`vs_verified_domains`) — דומיין, שיטה, מתי אומת, מתי פג, מספר סריקות.

### 4. יומן אירועים (Events)
היומן הגולמי מתוך `vs_audit_log`:
- זמן · אירוע · נתיב · כתובת שנסרקה · תוצאה · ציון · IP (hash בלבד).
- סינון לפי סוג אירוע + חיפוש בכתובת. עד 200 אירועים אחרונים.
- סוגי אירועים: `page_visit`, `scan_started`, `scan_completed`, `scan_failed`, `stage2_started`, `stage2_completed`, `badge_requested`, `terms_viewed`, `privacy_viewed`.

## צד שרת (Endpoints)

- **`GET /api/admin/dashboard`** — מחזיר את כל נתוני הסקירה + הסריקות + הלידים + הדומיינים המאומתים בקריאה אחת (אגרגציה בזיכרון, מבוסס על עד 5,000 רשומות אחרונות).
- **`POST /api/admin/dashboard`** — עדכון סטטוס ליד (`action: 'lead_status'`).
- **`GET /api/admin/logs`** — היומן הגולמי לטאב "יומן אירועים".
- **`api/_lib/admin-auth.ts`** — שכבת ההרשאה המשותפת לשני ה-endpoints (כדי שלא יסטו).

## מודל האבטחה (3 שכבות)

1. **Rate limit** לפי IP (30 בקשות/דקה).
2. **`ADMIN_SECRET`** — השוואה constant-time. סיסמה שגויה / חסרה → `404` (מסתירים שהראוט בכלל קיים).
3. **Cloudflare Turnstile** — חובה, אלא אם יש עוגיית session חתומה ב-HMAC (תוקף 15 דקות). עוברים את ה-Turnstile פעם אחת → מקבלים עוגייה שמספיקה לשני ה-endpoints.

הראוט עצמו מוחזר תמיד `Cache-Control: no-store` + `noindex`.

## לכידת לידים — איך זה עובד

- **טופס צור קשר** (`api/contact.ts`): שומר את הליד ל-DB **לפני** ששולח מייל, כך שגם אם המייל נכשל — הליד לא הולך לאיבוד.
- **אימות בעלות** (`api/verify.ts`): כל ניסיון אימות עם מייל תקין נשמר כליד (כולל אם אומת או לא).
- שתי הכתיבות הן fail-soft — לעולם לא שוברות את התגובה למשתמש.

> **שינוי לעומת קודם:** עד עכשיו טופס הקשר רק שלח מייל, וזרימת האימות לא שמרה את המייל בכלל — לידים אבדו. עכשיו הכל נשמר ונראה בלוח.
